💡 Como funciona
A música não fica no localStorage — fica salva separadamente no navegador (IndexedDB). Na hora de gerar o arquivo para deploy, o botão "Baixar pacote completo" na aba Deploy vai baixar um .zip com o index.html + o arquivo music.mp3 prontos. Assim não tem limite de espaço para a música! 🎵
🚀 Deploy no Netlify
Publique sua página gratuitamente
Gerar pacote completo

Clique abaixo para baixar um .zip com todos os arquivos prontos para o Netlify: index.html com dados embutidos + music.mp3 (se você fez upload).

Passo 1 — Extraia o .zip

Depois de baixar, extraia a pasta. Você verá: index.html, admin.html, music.mp3, netlify.toml.

Passo 2 — Deploy no Netlify
  1. Acesse netlify.com e crie uma conta grátis
  2. Clique em "Add new site" → "Deploy manually"
  3. Arraste a pasta inteira extraída para a área de deploy
  4. Aguarde ~30 segundos
  5. Copie o link e mande para a Lorrany! 💕
Passo 3 — URL personalizada (opcional)

Nas configurações do site no Netlify, renomeie para algo como lorrany-e-lucas.netlify.app.

Envia todos os dados (textos, fotos e música) para o repositório. Qualquer dispositivo que abrir o site verá as informações atualizadas.

Como configurar o GitHub Pages (primeira vez)
  1. Crie um repositório público no GitHub (ex: retrospectiva)
  2. Gere um token em Settings → Developer settings → Personal access tokens (classic) com escopo repo
  3. Preencha as credenciais acima e clique em Salvar credenciais
  4. Clique em Sincronizar com GitHub — isso cria os arquivos no repositório
  5. Faça upload manual dos arquivos index.html e admin.html para o repositório (uma única vez)
  6. No repositório → Settings → Pages → Source: Deploy from branch → Branch: main
  7. Aguarde ~1 minuto → acesse seu-usuario.github.io/retrospectiva 💕
